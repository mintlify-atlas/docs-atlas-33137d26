## List Servers

<CodeGroup>
```http GET /api/servers
GET /api/servers
Authorization: Bearer {token}
```
</CodeGroup>

Lists all servers. Users see their own servers, admins see all servers.

### Authentication

Requires authentication with valid session token.

### Response

<ResponseField name="servers" type="array">
  Array of server objects
  
  <Expandable title="Server Object">
    <ResponseField name="id" type="string">Server UUID</ResponseField>
    <ResponseField name="shortId" type="string">First section of UUID</ResponseField>
    <ResponseField name="name" type="string">Server name</ResponseField>
    <ResponseField name="description" type="string">Server description</ResponseField>
    <ResponseField name="status" type="string">INSTALLING | STARTING | RUNNING | STOPPING | STOPPED | SUSPENDED | MAINTENANCE | RESTORING | ERROR</ResponseField>
    <ResponseField name="memory" type="string">Memory limit in MiB (BigInt as string)</ResponseField>
    <ResponseField name="disk" type="string">Disk limit in MiB (BigInt as string)</ResponseField>
    <ResponseField name="cpu" type="number">CPU limit as percentage (100 = 1 core)</ResponseField>
    <ResponseField name="swap" type="string">Swap in MiB: -1 = unlimited, 0 = disabled, positive values = limited</ResponseField>
    <ResponseField name="node" type="object">Node information</ResponseField>
    <ResponseField name="blueprint" type="object">Blueprint information</ResponseField>
    <ResponseField name="owner" type="object">Owner information</ResponseField>
    <ResponseField name="allocations" type="array">Network allocations</ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

## Create Server

<CodeGroup>
```http POST /api/servers
POST /api/servers
Authorization: Bearer {token}
Content-Type: application/json
```
</CodeGroup>

Create a new server instance. Admin only.

### Authentication

Requires admin role.

### Request Body

<ParamField body="name" type="string" required>
  Server name (1-100 characters)
</ParamField>

<ParamField body="description" type="string">
  Server description
</ParamField>

<ParamField body="nodeId" type="string" required>
  Node UUID where server will be deployed
</ParamField>

<ParamField body="blueprintId" type="string" required>
  Blueprint UUID (game type)
</ParamField>

<ParamField body="ownerId" type="string">
  Owner user ID (defaults to current user)
</ParamField>

<ParamField body="memory" type="number" required>
  Memory limit in MiB
</ParamField>

<ParamField body="disk" type="number" required>
  Disk limit in MiB
</ParamField>

<ParamField body="cpu" type="number" required>
  CPU as percentage (100 = 1 thread)
</ParamField>

<ParamField body="cpuPinning" type="string">
  CPU pinning (e.g., "0,1,2,3" or "0-4")
</ParamField>

<ParamField body="swap" type="number" default="-1">
  Swap in MiB: -1 = unlimited, 0 = disabled, positive values = limited
</ParamField>

<ParamField body="oomKillDisable" type="boolean" default="false">
  Disable OOM killer
</ParamField>

<ParamField body="backupLimit" type="number" default="3">
  Maximum number of backups
</ParamField>

<ParamField body="allocationLimit" type="number" default="1">
  Maximum allocations user can add
</ParamField>

<ParamField body="allocationIds" type="array" required>
  Array of allocation IDs (at least one required)
</ParamField>

<ParamField body="config" type="object">
  Override blueprint config
</ParamField>

<ParamField body="variables" type="object">
  Override blueprint variables (key-value pairs)
</ParamField>

<ParamField body="dockerImage" type="string">
  Selected docker image from blueprint
</ParamField>

<ParamField body="autoShutdownEnabled" type="boolean">
  null = inherit global, true = enabled, false = disabled
</ParamField>

<ParamField body="autoShutdownTimeout" type="number">
  Inactivity timeout in minutes (1-10080)
</ParamField>

<RequestExample>
```json
{
  "name": "My Minecraft Server",
  "description": "Survival server",
  "nodeId": "node-uuid",
  "blueprintId": "minecraft-java",
  "memory": 2048,
  "disk": 10240,
  "cpu": 100,
  "swap": 512,
  "backupLimit": 5,
  "allocationLimit": 2,
  "allocationIds": ["alloc-1"],
  "variables": {
    "MINECRAFT_VERSION": "1.20.4",
    "SERVER_JARFILE": "server.jar"
  },
  "dockerImage": "ghcr.io/pterodactyl/yolks:java_21"
}
```
</RequestExample>

### Response

Returns the created server object with status 201.

### Errors

- `400` - Validation failed, node offline, insufficient resources, or invalid allocations
- `404` - Node or blueprint not found
- `500` - Daemon communication error

## Power Actions

### Start Server

<CodeGroup>
```http POST /api/servers/:serverId/start
POST /api/servers/{serverId}/start
Authorization: Bearer {token}
```
</CodeGroup>

Start the server.

### Stop Server

<CodeGroup>
```http POST /api/servers/:serverId/stop
POST /api/servers/{serverId}/stop
Authorization: Bearer {token}
```
</CodeGroup>

Gracefully stop the server using configured stop command.

### Restart Server

<CodeGroup>
```http POST /api/servers/:serverId/restart
POST /api/servers/{serverId}/restart
Authorization: Bearer {token}
```
</CodeGroup>

Restart the server.

### Kill Server

<CodeGroup>
```http POST /api/servers/:serverId/kill
POST /api/servers/{serverId}/kill
Authorization: Bearer {token}
```
</CodeGroup>

Forcefully kill the server process (SIGKILL).

### Response

All power actions return:

```json
{
  "success": true,
  "status": "STARTING" // or STOPPING, STOPPED
}
```

### Errors

- `400` - Node is offline or server is suspended
- `500` - Daemon communication error

POST /api/servers/{serverId}/change-blueprint
Authorization: Bearer {token}
Content-Type: application/json
