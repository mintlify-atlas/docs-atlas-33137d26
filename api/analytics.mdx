<Note>All analytics endpoints require admin authentication.</Note>

## Get System Metrics

Fetch current system-wide metrics.

<CodeGroup>
```bash cURL
curl -X GET https://api.stellarstack.io/analytics/system-metrics \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

### Response

<ResponseField name="totalServers" type="number">
  Total number of servers
</ResponseField>

<ResponseField name="totalUsers" type="number">
  Total number of users
</ResponseField>

<ResponseField name="activeConnections" type="number">
  Active WebSocket connections
</ResponseField>

<ResponseField name="averageCpuUsage" type="number">
  Average CPU usage percentage across all nodes
</ResponseField>

<ResponseField name="averageMemoryUsage" type="number">
  Average memory usage percentage
</ResponseField>

<ResponseField name="averageDiskUsage" type="number">
  Average disk usage percentage
</ResponseField>

<ResponseField name="uptime" type="number">
  API uptime in seconds
</ResponseField>

<ResponseField name="totalNodes" type="number">
  Total number of nodes
</ResponseField>

<ResponseField name="healthyNodes" type="number">
  Number of healthy nodes (heartbeat within last 5 minutes)
</ResponseField>

## Get Server Metrics

Fetch server resource metrics, optionally filtered by node.

<CodeGroup>
```bash cURL
curl -X GET "https://api.stellarstack.io/analytics/server-metrics?nodeId=node-id" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

### Query Parameters

<ParamField query="nodeId" type="string">
  Filter servers by node ID
</ParamField>

### Response

Returns an array of server metric objects (limited to 50 servers).

<ResponseField name="serverId" type="string">
  Server ID
</ResponseField>

<ResponseField name="serverName" type="string">
  Server name
</ResponseField>

<ResponseField name="status" type="string">
  Server status: "running", "stopped", "installing", "suspended", "error"
</ResponseField>

<ResponseField name="cpuUsage" type="number">
  CPU usage percentage
</ResponseField>

<ResponseField name="memoryUsage" type="number">
  Memory used in bytes
</ResponseField>

<ResponseField name="memoryLimit" type="number">
  Memory limit in bytes
</ResponseField>

<ResponseField name="diskUsage" type="number">
  Disk used in bytes
</ResponseField>

<ResponseField name="diskLimit" type="number">
  Disk limit in bytes
</ResponseField>

<ResponseField name="players" type="number" required={false}>
  Number of connected players (game servers only)
</ResponseField>

<ResponseField name="fps" type="number" required={false}>
  Frames per second (game servers only)
</ResponseField>

<ResponseField name="tps" type="number" required={false}>
  Ticks per second (Minecraft servers only)
</ResponseField>

<ResponseField name="uptime" type="number">
  Server uptime in seconds
</ResponseField>

<ResponseField name="lastUpdate" type="number">
  Unix timestamp of last metrics update
</ResponseField>

## Get CPU Time Series

Fetch CPU usage time series data.

<CodeGroup>
```bash cURL
curl -X GET "https://api.stellarstack.io/analytics/cpu-series?timeRange=7d" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

### Query Parameters

<ParamField query="timeRange" type="string" default="7d">
  Time range: "24h", "7d", "30d", "90d", "1y"
</ParamField>

### Response

<ResponseField name="dataPoints" type="array">
  Time series data points
  
  <Expandable title="data point object">
    <ResponseField name="timestamp" type="number">
      Unix timestamp
    </ResponseField>
    <ResponseField name="value" type="number">
      Average CPU usage percentage for this interval
    </ResponseField>
    <ResponseField name="label" type="string">
      Human-readable timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="average" type="number">
  Average CPU usage across all data points
</ResponseField>

<ResponseField name="min" type="number">
  Minimum CPU usage
</ResponseField>

<ResponseField name="max" type="number">
  Maximum CPU usage
</ResponseField>

<ResponseField name="peak" type="number">
  Peak CPU usage (same as max)
</ResponseField>

## Export Analytics

Export analytics data in CSV or JSON format.

<CodeGroup>
```bash cURL
curl -X GET "https://api.stellarstack.io/analytics/export?timeRange=7d&format=csv" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o analytics.csv
```

```bash JSON Export
curl -X GET "https://api.stellarstack.io/analytics/export?timeRange=7d&format=json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o analytics.json
```
</CodeGroup>

### Query Parameters

<ParamField query="timeRange" type="string" default="7d">
  Time range: "24h", "7d", "30d", "90d", "1y"
</ParamField>

<ParamField query="format" type="string" default="json">
  Export format: "csv" or "json"
</ParamField>

### Response

**CSV Format:**
- Content-Type: text/csv
- Formatted CSV with sections for system metrics, node metrics, server metrics, backups, blueprints, API, and webhooks

**JSON Format:**
- Content-Type: application/json
- Complete analytics dashboard data (same as Get Analytics Dashboard)

**400 Bad Request** - Unsupported format

<ResponseField name="error" type="string">
  "Unsupported format. Supported formats: csv, json"
</ResponseField>
