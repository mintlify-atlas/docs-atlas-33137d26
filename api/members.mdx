## Member Management

### List Members

<CodeGroup>
```http GET /api/servers/:serverId/members
GET /api/servers/{serverId}/members
Authorization: Bearer {token}
```
</CodeGroup>

List all members (subusers) of a server.

#### Path Parameters

<ParamField path="serverId" type="string" required>
  Server UUID
</ParamField>

#### Authentication

Requires server access and `users.read` permission.

#### Response

<ResponseField name="members" type="array">
  Array of member objects
  
  <Expandable title="Member Object">
    <ResponseField name="id" type="string">Member UUID</ResponseField>
    <ResponseField name="userId" type="string">User UUID</ResponseField>
    <ResponseField name="user" type="object">User information</ResponseField>
    <ResponseField name="permissions" type="array">Array of permission strings</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp</ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json
[
  {
    "id": "member-1",
    "userId": "user-123",
    "user": {
      "id": "user-123",
      "name": "John Doe",
      "email": "john@example.com",
      "image": "https://..."
    },
    "permissions": [
      "control.start",
      "control.stop",
      "control.restart",
      "console.read",
      "console.write",
      "files.read"
    ],
    "createdAt": "2024-01-15T12:00:00Z",
    "updatedAt": "2024-01-15T12:00:00Z"
  }
]
```
</ResponseExample>

### Update Member Permissions

<CodeGroup>
```http PATCH /api/servers/:serverId/members/:memberId
PATCH /api/servers/{serverId}/members/{memberId}
Authorization: Bearer {token}
Content-Type: application/json
```
</CodeGroup>

Update a member's permissions.

#### Path Parameters

<ParamField path="serverId" type="string" required>
  Server UUID
</ParamField>

<ParamField path="memberId" type="string" required>
  Member UUID
</ParamField>

#### Authentication

Requires server access and `users.update` permission.

#### Request Body

<ParamField body="permissions" type="array" required>
  Array of permission strings (at least one required)
</ParamField>

<RequestExample>
```json
{
  "permissions": [
    "control.start",
    "control.stop",
    "console.read",
    "console.write",
    "files.read",
    "files.write"
  ]
}
```
</RequestExample>

#### Response

Returns the updated member object.

#### Errors

- `400` - Validation failed (permissions array must have at least one item)
- `404` - Member not found

---

## Invitation Management

### List Invitations

<CodeGroup>
```http GET /api/servers/:serverId/invitations
GET /api/servers/{serverId}/invitations
Authorization: Bearer {token}
```
</CodeGroup>

List all pending invitations for a server.

#### Path Parameters

<ParamField path="serverId" type="string" required>
  Server UUID
</ParamField>

#### Authentication

Requires server access and `users.read` permission.

#### Response

<ResponseField name="invitations" type="array">
  Array of invitation objects (only pending, non-expired invitations)
  
  <Expandable title="Invitation Object">
    <ResponseField name="id" type="string">Invitation UUID</ResponseField>
    <ResponseField name="email" type="string">Invitee email address</ResponseField>
    <ResponseField name="permissions" type="array">Permissions to grant on acceptance</ResponseField>
    <ResponseField name="inviter" type="object">User who sent the invitation</ResponseField>
    <ResponseField name="expiresAt" type="string">ISO 8601 timestamp (7 days from creation)</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp</ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json
[
  {
    "id": "inv-1",
    "email": "newuser@example.com",
    "permissions": ["control.start", "console.read"],
    "inviter": {
      "id": "user-123",
      "name": "Server Owner",
      "email": "owner@example.com"
    },
    "expiresAt": "2024-01-22T12:00:00Z",
    "createdAt": "2024-01-15T12:00:00Z"
  }
]
```
</ResponseExample>

### Send Invitation

<CodeGroup>
```http POST /api/servers/:serverId/invitations
POST /api/servers/{serverId}/invitations
Authorization: Bearer {token}
Content-Type: application/json
```
</CodeGroup>

Send an invitation to a user to join the server.

#### Path Parameters

<ParamField path="serverId" type="string" required>
  Server UUID
</ParamField>

#### Authentication

Requires server access and `users.create` permission.

#### Request Body

<ParamField body="email" type="string" required>
  Email address to invite
</ParamField>

<ParamField body="permissions" type="array" required>
  Permissions to grant (at least one required)
</ParamField>

<RequestExample>
```json
{
  "email": "newuser@example.com",
  "permissions": [
    "control.start",
    "control.stop",
    "console.read",
    "console.write"
  ]
}
```
</RequestExample>

#### Response

<ResponseField name="id" type="string">
  Invitation UUID
</ResponseField>

<ResponseField name="email" type="string">
  Invitee email
</ResponseField>

<ResponseField name="permissions" type="array">
  Granted permissions
</ResponseField>

<ResponseField name="token" type="string">
  Invitation token (for manual sharing)
</ResponseField>

<ResponseField name="acceptUrl" type="string">
  Complete acceptance URL
</ResponseField>

<ResponseField name="expiresAt" type="string">
  Expiration timestamp
</ResponseField>

<ResponseField name="createdAt" type="string">
  Creation timestamp
</ResponseField>

<ResponseExample>
```json
{
  "id": "inv-2",
  "email": "newuser@example.com",
  "permissions": ["control.start", "control.stop", "console.read", "console.write"],
  "token": "unique-invitation-token",
  "acceptUrl": "https://stellarstack.io/servers/invitation/unique-invitation-token",
  "expiresAt": "2024-01-22T14:00:00Z",
  "createdAt": "2024-01-15T14:00:00Z"
}
```
</ResponseExample>

#### Errors

- `400` - User already a member, or pending invitation already exists

These endpoints are used by invited users to view and accept invitations.

### Get Invitation Details

<CodeGroup>
```http GET /api/servers/members/invitation/:token
GET /api/servers/members/invitation/{token}
```
</CodeGroup>

Get invitation details by token (for acceptance page). No authentication required.

#### Path Parameters

<ParamField path="token" type="string" required>
  Invitation token
</ParamField>

#### Response

<ResponseExample>
```json
{
  "id": "inv-1",
  "server": {
    "id": "server-uuid",
    "name": "My Minecraft Server"
  },
  "inviter": {
    "name": "Server Owner",
    "email": "owner@example.com"

  "expiresAt": "2024-01-22T12:00:00Z"
```

### Accept Invitation

<CodeGroup>
```http POST /api/servers/members/invitation/:token/accept
POST /api/servers/members/invitation/{token}/accept
Authorization: Bearer {token}
```
</CodeGroup>

Accept an invitation and become a server member.

#### Path Parameters

<ParamField path="token" type="string" required>
  Invitation token
</ParamField>

#### Authentication

Requires user to be logged in.

#### Response

```json
{
  "success": true,
  "server": {
    "id": "server-uuid",
    "name": "My Minecraft Server"
  }
}
```

#### Errors

- `400` - Already a member, already used/declined, expired, or user is the owner
- `401` - Not logged in
- `404` - Invitation not found

## Common Permission Nodes

Here are commonly used permission nodes:

### Control
- `control.start` - Start the server
- `control.stop` - Stop the server
- `control.restart` - Restart the server
- `control.kill` - Force kill the server

### Console
- `console.read` - View console output
- `console.write` - Send commands to console

### Files
- `files.read` - View and download files
- `files.write` - Create and edit files
- `files.delete` - Delete files
- `files.archive` - Create archives

### Backups
- `backups.read` - View backups
- `backups.create` - Create backups
- `backups.restore` - Restore backups
- `backups.delete` - Delete backups
- `backups.download` - Download backups

### Settings
- `settings.read` - View server settings
- `settings.update` - Modify server settings
- `settings.rename` - Rename server
- `settings.reinstall` - Reinstall server

### Users
- `users.read` - View members
- `users.create` - Invite members
- `users.update` - Update member permissions
- `users.delete` - Remove members
